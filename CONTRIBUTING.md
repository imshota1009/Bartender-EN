# Bartender Game への貢献について

まずは、貢献をご検討いただきありがとうございます！🎉

以下は、Bartender Game に貢献いただくためのガイドラインです。これらはあくまでガイドラインであり、厳格なルールではありません。最善の判断を行い、必要に応じてプルリクエストでドキュメントの変更をご提案ください。

## 行動規範

このプロジェクトおよび参加者全員は、当プロジェクトの行動規範に従うものとします。参加される皆様は、この規範を遵守していただきますようお願いいたします。

## 貢献方法

### バグ報告

バグを発見された場合は、以下のガイドラインに従って報告をお願いします。これにより、メンテナやコミュニティの他のメンバーが問題を理解し、再現し、解決策を見つける手助けとなります。

- **明確でわかりやすいタイトル** を付けてください。
- **問題を再現するための手順** をできるだけ詳細に記述してください。
- **具体的な例を提示** してください。関連するファイルやGitHubプロジェクトへのリンク、コードスニペットなどを含めてください。
- **どのような結果になったか**、そして本来期待していた結果とその理由を説明してください。
- **スクリーンショットやGIF動画** があると、問題の理解に大いに役立ちます。

### 機能改善の提案

新しい機能の提案や、既存機能の改善案については、以下のガイドラインに従ってください。

- **明確でわかりやすいタイトル** を付けてください。
- **提案内容の詳細な説明** を記述してください。
- **具体的な使用例** を提示してください。
- **現状の挙動** と **期待する挙動**、その理由を説明してください。

### プルリクエスト (PR)

1.  リポジトリをフォークし、`main` ブランチから作業用ブランチを作成してください。
2.  テストが必要なコードを追加した場合は、テストコードも追加してください。
3.  APIを変更した場合は、ドキュメントも更新してください。
4.  テストがすべて通ることを確認してください。
5.  コード規約（Lintなど）に従っていることを確認してください。
6.  プルリクエストを作成してください！

## スタイルガイド

### JavaScript

- モダンなJavaScript (ES6+) を使用してください。
- `var` ではなく `const` と `let` を使用してください。
- セミコロンを使用してください。
- 既存のコードのフォーマットに従ってください。

### HTML/CSS

- 可能な限り、汎用的なタグではなく、特定のクラスを使用してスタイリングしてください。
- セマンティックなHTML構造を維持してください。

## ライセンス
